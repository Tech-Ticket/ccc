Welcome to Session 8. You passed Level 1. You can build a layout fast. But can you read? Can you spot the traps?

In high-level competitions, judges intentionally write instructions that seem simple but contain hidden requirements and contradictory rules. If you code blindly without thinking, your layout will shatter when they test it.

  • You will build an interactive component from a highly deceptive prompt.
  • You must extract the TRUE requirements before coding.
  • The clock is your enemy. You have exactly 90 Minutes.

📜 The Official Rules (Read Carefully)

Failure to follow these constraints will result in immediate disqualification.

❌

NO Images Allowed

No <img> tags. No background URLs.

❌

NO JavaScript Allowed

All interaction (hovers, state changes) must be handled natively by CSS.

⚠️

STRICT Adherence Required

If the prompt implies a behavior (like "smoothness" or "equality"), you must code for it, even if it doesn't give you the exact CSS property.

🏆 The Challenge Execution

Build the "Interactive Info Cards Section" based on the prompt analyzed above.

Design Guidelines:

  • Base State: Dark card background (#1e293b), light text, blue button.
  • Hover State: Blue card background (#3b82f6), white text, dark button with white text.
  • Layout: Flexbox. Gap of 20px. Responsive (stack on mobile).
  • Number Badge: A small circle at the top left of each card containing "01", "02", "03". Use absolute positioning or flexbox to place it.

Submission Checklist

1. Equal Heights

Did you use align-items: stretch on the container so all cards are the exact same height?

2. Button Placement

Did you use margin-top: auto on the buttons so they are perfectly aligned at the bottom of the cards, regardless of text length?

3. Complex Hover Selection

Did you use .card:hover .btn { ... } so the button changes state when the CARD is hovered?

4. Smooth Transitions

Is there a transition property on the base classes so the color shift is smooth, not instant?

Official Scoring Weights

35% Requirements (Traps)
25% Layout Quality
20% Hover Effects
10% Responsive
10% Code Cleanliness
To fix the button alignment trap:
Make the card itself a flex container!
.card { display: flex; flex-direction: column; }
Then tell the button to push away from everything above it:
.btn { margin-top: auto; }

🔍 Judges' Review: Surviving the Traps

If you have finished, review the optimal solution below. Level 2 separates the amateurs from the professionals based on Attention to Detail.

The Optimal Solution Architecture

HTML Structure 
<div class="card-container">
  <!-- Card 1 -->
  <div class="card">
    <div class="badge">01</div>
    <h3>Starter Plan</h3>
    <p>This is a short description.</p>
    <button class="btn">Learn More</button>
  </div>
  
  <!-- Card 2 with a longer description to test the height trap -->
  <div class="card">
    <div class="badge">02</div>
    <h3>Pro Plan</h3>
    <p>This is a much longer description that will force this card to be taller than the others if we aren't careful with our flexbox rules!</p>
    <button class="btn">Learn More</button>
  </div>
</div>
CSS (Solving the Traps) 
/* Trap 1 Solution: Equal Heights & Responsive */
.card-container {
  display: flex;
  gap: 20px;
  flex-wrap: wrap;
  align-items: stretch; /* Forces equal height! */
}

/* Trap 2 Solution: Button Alignment & Smoothness */
.card {
  flex: 1;
  min-width: 250px;
  background: #1e293b;
  padding: 30px;
  
  /* Make card a flex container to push button down */
  display: flex;
  flex-direction: column;
  
  /* Smooth transition for the hover effects */
  transition: all 0.3s ease; 
}

.btn {
  margin-top: auto; /* Pushes button to bottom! */
  background: #3b82f6;
  transition: all 0.3s ease;
}

/* Trap 3 Solution: Advanced Hover Targeting */
.card:hover {
  background: #3b82f6; /* Card turns blue */
  color: white;
}

/* When card is hovered, change the button inside it! */
.card:hover .btn {
  background: #0f172a; color: white;
}
