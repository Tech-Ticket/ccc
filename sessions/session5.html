Session 5 — Responsive Design Under Pressure

🎯 Session Overview

Welcome to Session 5! You have learned how to build beautiful card layouts using Flexbox. But there is a massive trap waiting for you in competitions.

You code your layout perfectly on a 15-inch laptop screen. Time runs out. The judge opens your code on an iPad or a small laptop monitor... and your layout completely shatters. Cards overflow the screen, text gets cut off, and you lose points.

Today, you will learn to survive the screen-size test. You will learn:

  • What Responsive Design is and why judges look for it.
  • How to use Media Queries to change your CSS rules based on screen size.
  • How to train your mind to think "Mobile-First".
  • How to make Flexbox wrap elements automatically to prevent broken layouts.

🏆 The Competition Context

In a coding challenge, the prompt might not explicitly say "Make it responsive for mobile phones." But judges are developers. They will instinctively resize the browser window to see if your code breaks.

Common Layout Failures:

  • ❌ Fixed Widths: Hardcoding width: 1200px guarantees your site will break on small screens, causing a horrible horizontal scrollbar.
  • ❌ Squished Cards: Using Flexbox without wrapping forces cards to become incredibly thin, pushing text out of their borders.
  • ✅ The Winning Move: Using flexible widths (% or max-width) and media queries to cleanly stack elements vertically when space runs out.

📱 What is Responsive Design?

Responsive design is the practice of building a single HTML page that changes its layout and CSS styling depending on the width of the screen (the "viewport").

📏 The Secret Weapon: Media Queries

To tell the browser to change the layout, we use a CSS "At-Rule" called @media. Think of it as an "IF statement" for CSS.

CSS 
/* 1. Default Style (Mobile First Approach) */
body {
  background-color: #0f172a; /* Dark Blue */
}

/* 2. Media Query for larger screens (Tablet/Desktop) */
@media (min-width: 768px) {
  body {
    background-color: #10b981; /* Turns Green on Desktop! */
  }
}

Instead of building the Desktop site first and trying to fix it for mobile (which is hard), champions build the Mobile site first!

You write your default CSS so that everything stacks vertically (the natural way HTML flows). Then, you use @media (min-width: 768px) to say: "If the screen is wide enough, turn on Flexbox and put them side-by-side!"

🔄 Flexbox Wrapping & Flexible Widths

Sometimes you don't even need a media query! Flexbox has a built-in superpower called flex-wrap.

If you give your cards a min-width, and tell the container to flex-wrap: wrap, the browser will automatically drop cards to the next line if there isn't enough room.

CSS 
.card-container {
  display: flex;
  flex-wrap: wrap; /* Allows cards to move to the next row */
  gap: 20px;
}

.card {
  flex: 1;             /* Grow to fill empty space */
  min-width: 250px;  /* BUT never shrink smaller than 250px! */
}

🏋️‍♂️ Training Drills

Open your code editor. Build a fast, flexible layout.

Drill 1 (Easy): The Safe Container

Never let text touch the edge of the screen! Create a <div class="container">. In CSS, limit its size using max-width: 800px; and center it on the screen using margin: 0 auto;.

.container { max-width: 800px; margin: 0 auto; padding: 20px; }
Using max-width instead of width means it can shrink on small screens safely!

Drill 2 (Medium): Media Query Flip

Create a layout that is a single column by default (mobile-first). Add a media query for min-width: 600px. Inside the media query, change the container to display: flex; so it becomes a row on larger screens.

.row { display: block; } /* Mobile */
@media (min-width: 600px) { .row { display: flex; } } /* Desktop */

Drill 3 (Hard): The 100% Image Fix

Add a massive image to your HTML inside a small div. It will break the layout. Fix it by applying the golden rule of responsive images in your CSS.

Always include this in your CSS:
img { max-width: 100%; height: auto; }

🔥 Competition Task: Responsive Card Section

Brief: "Build an info section with 3 cards that looks great on both mobile and desktop."

Requirements:

  1. Create a container holding 3 simple cards (Title, Description, Button).
  2. Mobile View (Default): The cards must be stacked vertically (one on top of the other).
  3. Desktop View: When the screen is wider than 768px, use a Media Query to align the cards side-by-side using Flexbox.
  4. Cards must have clean padding, background colors, and a gap between them.
/* 1. Mobile First: Default Styles */
.card-container {
  display: flex;
  flex-direction: column; /* Stack vertically on phones */
  gap: 20px;
}

.card {
  background: #1e293b;
  padding: 20px;
  border-radius: 8px;
}

/* 2. Desktop Styles: Only apply if screen is wide enough */
@media (min-width: 768px) {
  .card-container {
    flex-direction: row; /* Put side-by-side on desktop */
  }
  
  .card {
    flex: 1; /* Make them equal width */
  }
}

📝 Session Summary

  • Responsive Design ensures your layout doesn't break when judges test your code on different devices.
  • Media Queries (@media) allow you to inject specific CSS rules only when the screen matches your size conditions.
  • Mobile-First Thinking: Write the default CSS for stacked, simple mobile layouts first. Then use min-width queries to add complex Flexbox rows for desktops.
  • Flex Wrap: Using flex-wrap: wrap; with a min-width on child items is a genius way to create responsive layouts without writing a single media query!
