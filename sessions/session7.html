🏆 The Challenge Brief

TASK: Build a "Sports Training" Landing Section.

REQUIREMENTS:
1. The section must have a main title: "Elite Athlete Programs".
2. Below the title, there must be exactly THREE identical cards in a single row.
3. The cards are for: "Strength Training", "Speed & Agility", and "Endurance".
4. Each card MUST contain:
    a) A circular badge at the top containing a relevant Unicode Icon.
    b) The Program Title.
    c) A short description paragraph.
    d) A "Join Now" button at the bottom.
5. Design: The layout must use Flexbox. Cards must have a dark background, white text, 20px padding, rounded corners, and a subtle border.
6. Responsiveness: On screens smaller than 768px, the cards must stack vertically.
7. Interaction: When hovering over a card, it should lift up slightly and gain a colored box-shadow.
Coach's Advice:
1. Build the HTML skeleton first. Don't touch CSS until all text and buttons are on the page.
2. Use display: flex; gap: 20px; on the parent container.
3. Use flex: 1; min-width: 250px; flex-wrap: wrap; on the cards. This makes them responsive automatically without needing complex media queries!
4. Unicode ideas: 💪 (Strength), ⚡ (Speed), 🏃 (Endurance).

📋 Submission Checklist & Scoring

Before you declare you are finished, verify your work against the Judges' rubric.

1. Architecture Complete

Main title and 3 cards exist. No missing text or buttons.

2. Layout Accuracy

Cards are in a horizontal row on desktop and take up exactly equal width.

3. Constraint Adherence

4. Responsive Behavior

When the browser window is shrunk to mobile size, the cards stack vertically without text overflowing.

5. Polish & Interaction

Hovering over the cards triggers a smooth CSS transition (transform & box-shadow).

🔍 Judges' Review & Solution

If you have finished, or the timer has expired, review the optimal solution below. How does your code compare?

Common Mistakes Made Under Pressure:

  • Forgetting box-sizing: border-box; causing padding to blow up the card widths.
  • Putting the transition property on the :hover state instead of the base .card class (causing a jerky animation when the mouse leaves).
  • Writing completely separate media queries instead of just using flex-wrap and min-width.

The Optimal Speed-Code Solution

index.html 
<section class="programs">
  <h2 class="main-title">Elite Athlete Programs</h2>
  
  <div class="card-container">
    <!-- Card 1 -->
    <div class="card">
      <div class="icon-badge">💪</div>
      <h3>Strength Training</h3>
      <p>Build raw power and muscle.</p>
      <button class="btn">Join Now</button>
    </div>
    
    <!-- Duplicate card div twice for Speed/Endurance -->
  </div>
</section>
style.css (Layout & Logic) 
* { box-sizing: border-box; margin: 0; }

/* The Magic Responsive Layout */
.card-container {
  display: flex;
  gap: 20px;
  flex-wrap: wrap; /* Allows stacking on mobile */
  justify-content: center;
}

.card {
  flex: 1;             /* Forces equal width */
  min-width: 280px;    /* Forces stacking when screen is small! */
  background: #1e293b;
  padding: 30px;
  border-radius: 12px;
  transition: transform 0.3s, box-shadow 0.3s; /* Base state transition */
}

/* Hover Polish */
.card:hover {
  transform: translateY(-10px);
  box-shadow: 0 15px 30px rgba(0,0,0,0.5);
}

/* Fake Image Trick */
.icon-badge {
  width: 60px; height: 60px;
  border-radius: 50%;
  background: #3b82f6;
  display: flex;
  justify-content: center;
  align-items: center;
  font-size: 30px;
  margin-bottom: 20px;
}
