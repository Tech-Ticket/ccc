Session 4 — Constraints & Smart Problem Solving

🚨 Session Overview

Welcome to Session 4. You know HTML, CSS, Flexbox, Gradients, and Hover effects. You have the skills to build almost anything.

So why do talented students lose coding competitions?

They lose because they don't read the rules. They build a beautiful website, but they use an image when the rules said "No Images". Or they build 4 cards when the prompt asked for 3.

In this session, you will learn how to win by playing smart. You will learn to:

  • Understand the strict Constraints of coding competitions.
  • Read a problem statement and extract the exact requirements.
  • Use the READ → BREAK → PLAN → BUILD method.
  • Build complex UIs using clever workarounds (like Unicode icons and CSS shapes) when images are banned.

📜 Real Competition Rules

When you sit down at a competition, you will usually be given a piece of paper or a PDF with a "Design Brief". It will contain strict rules. If you break them, you get Zero Points for that section.

The Most Common Constraints:

  • ❌ No Images Allowed: No <img> tags. No background images from Google. Why? Because the judges want to test your CSS skills, not your ability to search for pictures.
  • ❌ No External Libraries: No Bootstrap, no Tailwind, no FontAwesome. Just pure, vanilla HTML and CSS.
  • ❌ No JavaScript: Unless it's a JS-specific competition, interactivity must be handled by CSS :hover, :active, or :focus.
  • ✅ Exact Match Required: If the prompt says "A dark blue background", do not use light blue because you think it looks better. Follow the instructions exactly.

🛠️ Clever Workarounds (Code Examples)

Since we can't use images or external icon libraries, we have to fake it using HTML and CSS.

Example 1: Using Unicode as Icons

Unicode characters are just text, so they are perfectly legal in "No Image" competitions. You can size them and color them just like normal fonts!

HTML & CSS 
<!-- HTML -->
<div class="icon">&#9889;</div> <!-- Lightning Bolt Unicode -->

/* CSS */
.icon {
  font-size: 48px;      /* Makes the icon huge */
  color: #f59e0b;       /* Colors the icon yellow */
  text-align: center;
}

Example 2: Faking an Avatar Image with CSS

If the prompt asks for a "user profile avatar" but bans images, you build a colored circle with the user's initials inside.

HTML & CSS 
<!-- HTML -->
<div class="avatar">JD</div> <!-- Initials for John Doe -->

/* CSS */
.avatar {
  width: 80px;
  height: 80px;
  border-radius: 50%; /* Makes it a perfect circle */
  background: linear-gradient(135deg, #3b82f6, #06b6d4);
  
  /* Center the initials perfectly inside the circle */
  display: flex;
  justify-content: center;
  align-items: center;
  color: white;
  font-weight: bold;
}

Flexbox isn't just for big page layouts! Using display: flex; justify-content: center; align-items: center; on a small container (like an 80x80px circle) is the absolute easiest, most reliable way to perfectly dead-center text or an icon inside a shape. No messy padding or line-height math required!

🏋️‍♂️ Training Drills

Open your code editor. Read the requirements carefully and build exactly what is asked.

Drill 1 (Easy): The Fake Image

Prompt: "Create a square product thumbnail (150x150px). Do not use images. The thumbnail should have a dark gray background and display the Unicode character for a camera (📷) in the center."

Create a div. Give it width/height 150px, background: #333;. Put &#128247; inside. Use Flexbox to center it, and make the font-size large (e.g., 50px).

Drill 2 (Medium): Plan the Layout

Prompt: "Design a navbar with a logo on the far left, and three links on the far right."
Task: Don't code it. Just write down the 3 CSS properties you need to apply to the parent container to achieve this layout.

1. display: flex;
2. justify-content: space-between;
3. align-items: center;

Drill 3 (Hard): Strict Constraints

Prompt: "Create a button. When hovered, the background color must invert, and the button must grow by exactly 10%. Smooth the transition to half a second."

Base button needs: transition: all 0.5s ease;
Hover state needs: transform: scale(1.1); and swap the background/text colors.

🔥 Competition Task: "No-Image" Sports Cards

This is a simulated competition task. Read the requirements carefully. Missing a requirement means losing points.

Brief: "Build a Sports Card Section displaying 3 sports."

Strict Requirements:

  1. Layout: Must be a single row of 3 cards using Flexbox.
  2. Constraints: NO IMAGES ALLOWED. NO EXTERNAL FONTS OR ICONS ALLOWED.
  3. Cards: Must be for Golf, Baseball, and Football.
  4. Card Content: Every card MUST contain:
    • A circular avatar at the top containing a Unicode icon representing the sport.
    • A Title (e.g., "Golf").
    • A short description paragraph.
    • A "Read More" button.
  5. Styling: The cards must have rounded corners and a box-shadow.

Check your structure against this. Did you use Flexbox to center the icon inside the avatar circle?

<!-- Structure for ONE of the cards -->
<div class="card-container"> <!-- Parent Flexbox -->

  <div class="sport-card">
    <!-- Fake Image (Circle with Flexbox centering) -->
    <div class="avatar">&#9917;</div> <!-- Golf Unicode -->
    
    <h3>Golf</h3>
    <p>Master the green.</p>
    <button class="btn">Read More</button>
  </div>
  
  <!-- ... other 2 cards ... -->
</div>

🏠 Homework: Deconstruct a Design

Go to a website you use often (like YouTube, Netflix, or a gaming site).

  • Pick one UI element (like a video thumbnail card or a user profile banner).
  • Try to rebuild it in your code editor using ONLY HTML and CSS.
  • If the element has an image, replace it with a colored div or a Unicode icon.
  • Focus on getting the padding, margins, and flexbox layout exactly right.

📝 Session Summary

  • Constraints are the game: Competitions restrict tools (like images or JS) to test your raw HTML/CSS logic.
  • Read → Break → Plan → Build: Never start coding immediately. Spend 3 minutes highlighting requirements and sketching the flexbox layout.
  • Fake it with CSS: Use Unicode characters (&#128640;), border-radius: 50% for circles, and gradients to simulate graphics when images are banned.
  • Micro-Flexbox: Flexbox isn't just for big layouts. Using display: flex; justify-content: center; align-items: center; is the ultimate hack for perfectly centering an icon inside a small circle or button.
